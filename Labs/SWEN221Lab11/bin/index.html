Welcome to the SimpleWeb HTTP Server!

If you've managed to get this far, then you've obviously setup and run the web server correctly --- well done!

SimpleWeb is really the most simple kind of web server possible, and was written by David J. Pearce on 2nd June, 2010 at the School of Engineering and Computer Science, Victoria University. SimpleWeb is not an HTTP 1.0/1.1 compliant server --- but it does at least serve up files if you ask it.

The main problem with SimpleWeb is that it is not multi-threaded. This means it can only handle one connection at a time. You can test this by trying to load a large file (like this one) in a separate tab and then, whilst this is progressing, following another link (like this one). What you should find, is that you won't be able to load any more pages from the webserver until that large file has completed (because it processes them one at a time). Obviously, if one user downloading a file meant all other users of the same website had to wait, the internet would be a very frustrating place indeed!

So, your job with this assignment is to update SimpleWeb to be multi-threaded!
